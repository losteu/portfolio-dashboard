# Портфельный анализ

Статический веб-дашборд для анализа продаж, товарных групп, клиентов, продукции и участков.

## Структура проекта

- `index.html` — входная точка
- `styles.css` — весь визуальный слой
- `app-core.js` — state, helpers, общие утилиты
- `app-data.js` — загрузка и подготовка данных
- `app-render.js` — рендер интерфейса
- `app.js` — инициализация и события
- `data/` — опциональная автозагрузка файлов при старте

Проект остается обычным статическим сайтом. То, что логика разбита на несколько файлов, не мешает публикации на GitHub Pages, Vercel, Netlify и любом другом static hosting.

## Как запускать локально

Есть два варианта:

1. Открыть `index.html` в браузере и загрузить файлы вручную через интерфейс.
2. Запустить через локальный сервер, если нужна автозагрузка из папки `data/`.

Пример простого локального сервера:

```powershell
cd C:\Users\user\Desktop\Анализ_ТГ_участков
python -m http.server 8000
```

После этого открыть `http://localhost:8000`.

## Какие данные можно использовать

### Продажи

Поддерживаются два варианта:

- `SQL: сырье / норматив / транспорт`
- `Продажи: фин. доход / себестоимость`

### Участки

Excel-файл участков нужен для:

- прямых затрат

## Публикация в GitHub Pages

Если хочешь открыть дашборд по ссылке и запускать его где угодно, можно выложить проект как обычный статический сайт.

### 1. Создать репозиторий и отправить проект

```powershell
cd C:\Users\niyazov.ad\Desktop\Анализ_ТГ_участков
git init
git branch -M main
git add .
git commit -m "Initial dashboard publish"
git remote add origin https://github.com/<your-user>/<your-repo>.git
git push -u origin main
```

### 2. Включить GitHub Pages

На GitHub:

1. Открыть `Settings`
2. Перейти в `Pages`
3. Включить `Deploy from a branch`
4. Выбрать ветку `main`
5. Выбрать папку `/ (root)`
6. Сохранить

После этого GitHub даст ссылку вида:

```text
https://<your-user>.github.io/<your-repo>/
```

## Автозагрузка данных на GitHub Pages

Приложение пытается автоматически загрузить:

- `data/sql_data.csv`
- `data/areas.xlsx`

Если эти файлы лежат в репозитории, сайт откроется уже с данными.

Если файлов в `data/` нет, сайт все равно будет работать, просто данные нужно загрузить вручную через кнопки в интерфейсе.

## Важное замечание по данным

Если данные коммерческие или внутренние, не выкладывай их в публичный репозиторий.

Безопасный вариант:

- публиковать код без файлов из `data/`
- после открытия сайта загружать реальные файлы вручную через UI
