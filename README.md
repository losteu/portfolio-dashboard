# Портфельный анализ — What-if дашборд

Интерактивный дашборд для ступенчатого маржинального анализа производственного ассортимента. Позволяет моделировать сценарии «что если закрыть участок / убрать товарную группу / отказать клиенту» и мгновенно видеть влияние на финансовый результат.

## Возможности

- **Ступенчатый P&L** — 4 уровня маржи: переменные → прямые участков → распределённые с логикой → общехозяйственные
- **What-if сценарии** — отключение участков, товарных групп, клиентов с мгновенным пересчётом
- **Фильтр по периоду** — выбор дат, затраты участков агрегируются только за выбранные месяцы
- **Анализ участков** — загрузка %, объём выпуска, три типа затрат, финансовый результат
- **Анализ клиентов** — рентабельность, доля в выручке, количество товарных групп
- **Загрузка данных** — CSV/XLSX из SQL + Excel по участкам

## Быстрый старт

1. Откройте `index.html` в браузере (двойной клик) или через GitHub Pages
2. Дашборд загрузится с демо-данными
3. Загрузите свои данные кнопками в шапке

## Источники данных

### 1. SQL данные (кнопка «SQL данные»)

CSV или XLSX выгрузка из SQL-скрипта маржинального анализа. Принимает разделители: `,` `;` `TAB`. Кодировки: UTF-8, Windows-1251.

**Колонки:**

| Колонка | Описание |
|---|---|
| `Код заявки` | ID заявки/отгрузки |
| `Дата заявки (отгрузки)` | Дата в любом формате |
| `Клиент` | Название клиента |
| `Товарная группа` | Группа готовой продукции |
| `Готовая продукция (ГП)` | Название продукта |
| `Код_ГП` | ID продукта (связь с участками) |
| `Отгружено ГП (кг)` | Количество в кг |
| `Выручка по ГП (RUR)` | Выручка в рублях |
| `Прямые_Затраты_Сырье_Склад` | Затраты на сырьё |
| `Прямые_Затраты_Нормы_Цех` | Производственные нормативы |
| `Прямые_Затраты_Транспорт` | Транспортные затраты |

### 2. Участки (кнопка «Участки .xlsx»)

Excel-файл с двумя листами.

**Лист 1 — «Участки»** (одна строка = один участок × один месяц):

| Колонка | Описание | Пример |
|---|---|---|
| `Наименование_участка` | Название участка | Глазурь Калуга |
| `Месяц` | Период (YYYY-MM) | 2025-11 |
| `Загруженность` | Процент загрузки (0–100) | 73.83 |
| `Прямые_затраты` | Прямые затраты за месяц | 5470507 |
| `Распред_с_логикой` | Распределённые с логикой | 2927991 |
| `Распред_без_логики` | Общехозяйственные | 1656497 |
| `Объем_выпуска_кг` | Выпуск продукции в кг | 45000 |
| `Маржинальный_доход` | Маржинальный доход | 9559851 |

**Лист 2 — «Привязка_ГП»:**

| Колонка | Описание | Пример |
|---|---|---|
| `Код_ГП` | ID продукта из SQL | 12345 |
| `Наименование_участка` | Точное название участка | Глазурь Калуга |

## Логика расчётов

```
Выручка (из SQL)
 − Сырьё, нормативы, транспорт (из SQL)
 = Маржа 1 (Contribution Margin)
 − Прямые затраты участков (из Excel, исчезают при закрытии)
 = Маржа 2
 − Распределённые с логикой (из Excel, частично перераспределяются)
 = Маржа 3
 − Распределённые без логики / общехозяйственные (из Excel, остаются)
 = Операционный результат
```

### Сценарий «закрыть участок»

При отключении участка:
- **Прямые затраты** — убираются полностью (реально исчезают)
- **Распределённые с логикой** — убираются (но в реальности могут частично лечь на другие участки)
- **Общехозяйственные** — убираются из расчёта участка (но в реальности остаются и перераспределяются)

Дашборд показывает чистый эффект: сколько выручки теряем vs сколько затрат экономим.
